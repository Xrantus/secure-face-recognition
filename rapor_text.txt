ABSTRACT  
 
Access control and security are fundamental requirements for modern organizations, yet traditional authentication 
mechanisms pose a critical threat to data security, as they remain vulnerable to loss, theft, and unauthorized 
duplication. The primary purpose of this project is to address these deficiencies by developing a robust, contactless, 
and offline-capable facial recognition system designed for fully local real-time surveillance and secure access 
management. 
 
To achieve this objective, the project methodology employs a distributed edge-server architecture that integrates 
deep learning algorithms within a fully local network environment. Unlike conventional cloud-dependent systems 
that suffer from high latency and external connectivity reliance, this solution offloads critical image processing tasks 
to autonomous embedded edge units. The system utilizes state-of-the-art models for face detection and high-
dimensional feature extraction (embedding generation) at the edge, while cosine similarity metrics are performed 
on a local backend server for precise identity verification. A rigorous data integrity pipeline is implemented, 
incorporating automatic Face Image Quality Assessment (FIQA) to filter unsuitable images during enrollment, 
ensuring that only high-quality data enters the recognition database. 
 
From a project management and system design perspective, the solution is structured into three cohesive layers: 
a mobile interface for administrative workflows and real-time alerts, a backend server for logic and database 
management, and edge nodes for continuous video analysis. This modular approach facilitates scalable 
deployment and efficient maintenance. The system includes comprehensive administrative controls, such as 
whitelist/blacklist management and audit logging, to create a fully verifiable security environment. 
 
The widespread impact of this work lies in its ability to democratize high-security surveillance by keeping the data 
in local. By successfully balancing computational efficiency with recognition accuracy, the proposed system offers 
a scalable, low-latency alternative to expensive commercial solutions. It significantly enhances operational 
efficiency for residential and commercial facilities by automating entry logs and providing immediate notifications 
for unauthorized access attempts, thereby eliminating human error in security protocols. 
 
Keywords:  Face Recognition, Edge Computing, Access Control, Deep Learning, Real-time Surveillance. 
 
 
1. PROJECT PURPOSE, INNOVATIVE ASPECTS AND TECHNOLOGICAL VALUE  
 
1.1. Purpose of the Project 
 
Existing access control infrastructures primarily rely on physical tokens -such as ID cards and key fobs- or manual 
verification processes. However, industry research indicates that these methods are inherently prone to operational 
risks. According to recent security reports, a significant percentage of security breaches originate from lost or stolen 
credentials, highlighting the vulnerability of token-based authentication [26]. Physical cards are easily duplicated, 
and traditional systems often fail to prevent "tailgating" which is a common security violation where unauthorized 
individuals gain access by following authorized personnel through a secure door  [27]. 
 
Beyond these vulnerabilities, traditional systems function as passive recording tools rather than active security 
agents, lacking the real-time responsiveness required to mitigate threats as they occur. The primary purpose of this 
project is to eliminate these dependencies and security gaps by developing a "Robust Face Recognition for 
Surveillance Cameras" system an intelligent solution that transforms standard video feeds into automated, 
contactless access control mechanisms. 
 
The concrete goals of this project are: 
 To eliminate physical dependencies: Replacing physical keys and cards with biometric facial embeddings. 
As noted in biometric security studies, physiological characteristics provide a higher level of non-
repudiation compared to possession-based tokens, preventing unauthorized access due to lost, stolen, or 
shared credentials [28]. 
 
 
 3 
  
  
 To reduce system latency:  Implementing a distributed edge-computing architecture  using Raspberry 
Pi devices to perform local face detection and feature extraction, thereby minimizing network bandwidth 
usage and ensuring rapid response times. 
 
 To ensure data integrity:  Establishing a comprehensive operational workflow that includes automatic 
image quality checks (FIQA) and strict administrative approval mechanisms (Whitelist/Blacklist) to prevent 
poor-quality data from compromising the system. 
 
 To provide scalable surveillance:  creating a system capable of 1:N identification (one-to-many 
searching) across multiple access points, scalable to millions of embeddings using modern vector 
database technologies. 
 
1.2 Innovative Aspects and Technological Value 
    
The proposed project distinguishes itself from existing solutions through its Edge Architecture  and comprehensive 
End-to-End Surveillance Pipeline . While traditional systems often rely on external cloud processing or raw 
algorithms, our innovation lies in bridging the gap between high-accuracy deep learning models and practical, 
privacy-centric operational requirements within a local network. 
 
The innovative aspects and technological superiority of the project are detailed below in comparison with similar 
national and international systems: 
 
1.2.1. Comparison with Academic Literature (FaceNet & ArcFace):  While state-of-the-art algorithms like 
FaceNet [1] and ArcFace [2] focus primarily on embedding accuracy, they often lack the necessary operational 
frameworks for real-world deployment. 
 
 Our Innovation:  We extend these algorithms by integrating an Automatic Quality Assurance (FIQA) layer, 
utilizing approaches similar to FaceQnet [4]. Unlike raw implementations, our system filters unsuitable 
images (blur, poor lighting, occlusion) before processing, ensuring high data integrity. Furthermore, we 
wrap the recognition engine in a managed workflow with whitelist/blacklist authorization, transforming a 
raw algorithm into a usable security product. 
 
1.2.2. Comparison with Commercial Mobile Applications:  Commercial solutions, such as FaceIntel Access 
Control [21] or Kisi Mobile Access [25], typically depend heavily on continuous internet connectivity and external 
servers for processing. This reliance introduces latency, privacy concerns regarding biometric data storage, and 
vulnerability to network failures. 
 
 Our Innovation (Edge Computing):  Our system introduces a distributed architecture based on Edge 
Computing principles [8] [16] , where the Edge Node (Raspberry Pi) performs local face detection and 
embedding generation. 
 
 By processing video frames at the edge and transmitting only vector data (embeddings) to the local 
backend, we eliminate the need for cloud dependency. This decentralized architecture minimizes 
bandwidth usage and ensures rapid response times within the local environment, allowing the system to 
operate securely in a completely offline setting. 
 
1.2.3. Comparison with Authentication Systems:  Most market-available applications, like BioID [22], focus 
primarily on 1:1 user authentication (verifying a claimed identity for a specific login attempt). 
 
 Our Innovation (1:N Surveillance):  Leveraging advanced frameworks like InsightFace [11] and 
RetinaFace [12], our system is engineered for 1:N identification. It is capable of scanning a live video 
stream to identify "Unknown" or "Restricted" individuals without active user interaction. Unlike 
authentication apps that provide simple grant/deny decisions, our system offers a categorized response 
mechanism (Authorized, Restricted, Unknown) that triggers real-time push notifications to security 
personnel. 
 
Technological Output:  The final product delivers a scalable, privacy-compliant security ecosystem supporting both 
Relational and Vector databases, capable of managing high-dimensional data efficiently for continuous surveillance 
operations completely offline.

 1.2.4. Robustness through Multi-Perspective Analysis  
Traditional face recognition systems often fail when presented with non-frontal faces or varying angles, as single-
image accuracy degrades significantly under unconstrained poses. 
 
 Our Innovation (Multi-Frame Processing):  To ensure robustness during real-time surveillance , our 
system does not rely on a single static snapshot. Instead, it processes a continuous video stream, 
capturing facial features from multiple perspectives  as the subject moves through the camera's field of 
view. By analyzing consecutive frames, the system selects the optimal angles to match against the stored 
references. This approach leverages the temporal information inherent in video sequences, as highlighted 
in Wang et al. [29] , to significantly reduce false negatives caused by momentary pose variations or 
occlusions.

METHODOLOGY  
 
This project implements a robust facial recognition system for surveillance by integrating deep learning algorithms, 
embedded edge computing, and a mobile application interface. The methodology follows a multi-tiered pipeline that 
encompasses data collection, feature extraction, classification, and real-time notification. The system utilizes the 
InsightFace library for generating high-dimensional facial embeddings and employs Cosine Similarity metrics for 
identity verification. To ensure efficient processing, the solution leverages YOLO v11 for person detection and 
classifiers for facial localization, distributed across edge devices and backend servers. 
 
2.1 System architecture and design (WP#1-2-3-4) 
 
The system design goes beyond a standalone mobile application; it is a distributed architecture where mobile 
interfaces, backend servers, and embedded edge devices communicate continuously to execute the security 
pipeline. To clarify the interactions between these components and the human operators, the system actors and 
their specific privileges are defined below, followed by an explanation of the operational pipeline. 
 
2.1.1 Operational Pipeline and Component Interaction 
 
The use cases presented in this section rely on a synchronized pipeline that connects the edge, backend, and 
mobile layers. The interaction flow is designed to minimize latency and ensure data consistency: 
 
 Edge Processing (Data Acquisition):  The operational pipeline begins at the embedded edge device 
(Raspberry Pi), which utilizes threaded frame capture to optimize performance. The device performs local 
face detection and generates embeddings using the deployed deep learning models, significantly reducing 
bandwidth usage by sending only vector data rather than raw video streams. 
 
 Backend Verification (Analysis):  The backend receives the facial embeddings and compares them 
against stored references in the database using Cosine Similarity calculations. Based on adjustable 
thresholds, the system classifies the detected individual as "Authorized," "Restricted," or "Unknown". 
 
 Mobile Synchronization (Action):  The results are pushed to the mobile application via secure API 
endpoints. If a security threat or unknown face is detected, the pipeline triggers an immediate push 
notification to the Admin, closing the loop between the autonomous edge detection and human decision-
making.

 2.1.2 System Actors and Privileges 
 
The system implements Role-Based Access Control (RBAC) to manage security and operational workflows. The 
defined actors and their respective privileges are as follows: 
 
 Admin: The primary overseer of the system with full access privileges. 
o Privileges:  The Admin is responsible for reviewing and approving pending user registration 
requests and assigning them to the Whitelist or Blacklist. They have the authority to view the full 
security dashboard, monitor device status, delete users from lists, and review detailed security 
activity logs. Additionally, the Admin receives and manages real-time security alerts for unknown 
or blacklisted detections. 
 
 Manager:  An administrative role with supervisory responsibility over a specific subset of users. 
o Privileges:  The Manager has access to a dedicated dashboard that displays statistics and recent 
activity specifically related to the users under their supervision ("My Users"), rather than the entire 
database. 
 
 User: The end-user who requires access to the facility. 
o Privileges:  The User interacts with the mobile application to register their identity. They are 
authorized to upload profile information and capture the required facial angles (Center, Left, 
Right) for system enrollment. They can also log in to view their profile status. 
 
 Edge Node (Actor):  The autonomous component deployed on the Raspberry Pi at the physical access 
points. 
o Privileges:  Unlike human actors, the Edge Node operates continuously as a system agent. It 
has the authority to capture video streams, execute local deep learning models for face detection 
and feature extraction, and initiate secure verification requests to the backend server. It acts as 
the bridge between the physical environment and the central database.

2.3 Dataset selection and preparation(WP#3-5-6) 
 
For the experimental evaluation of the system, the VGGFace2 [23] dataset was selected due to its extensive intra-
class variations in pose, illumination, and age, which are critical for simulating real-world surveillance conditions. 
To construct the facial recognition database (enrollment), a subset of 150 distinct identities was utilized. Unlike 
controlled environments that rely on specific facial angles, images were selected randomly for each identity to test 
the system's robustness against unconstrained inputs. The system's 1:N identification performance was 
subsequently validated using a train group consisting of 50 individuals from this enrolled dataset. 
 
 
2.4 Detection Algorithm and Model Architecture (WP#6) 
The core of the facial recognition system is built upon a multi-stage deep learning pipeline leveraging the 
InsightFace framework [11] , which prioritizes inference efficiency for real-time surveillance. The pipeline consists 
of the following synchronized stages: 
1. Face Detection and Alignment:  The process begins with YOLOv11 for efficient person detection to filter 
irrelevant frames. Subsequently, the system utilizes the RetinaFace algorithm [12] (embedded within the 
buffalo_l model pack) to precisely locate faces and landmarks. To optimize performance for edge 
processing, the input resolution is dynamically constrained (e.g., det_size=640x640), significantly reducing 
the computational load per frame. 

 
 
 8 
  
  
2. Face Image Quality Assessment (FIQA):  Before feature extraction, a quality control layer evaluates the 
aligned face images. As extensively categorized in the fundamental survey by Schlett et al. [30] ensuring 
high biometric utility is critical for reliable recognition. Accordingly, inputs that fall below a predefined quality 
threshold due to motion blur, extreme angles, or poor lighting are automatically discarded. This ensures 
that only high-fidelity data enters the recognition engine, minimizing false prediction rates. 
 
3. Feature Extraction:  High-quality aligned face images are then processed by an ArcFace model [2] (based 
on a ResNet backbone). This model maps the facial features into a 512-dimensional vector space, similar 
to techniques discussed in [1] and [13]. As demonstrated in the system's verification modules, the resulting 
embeddings are normalized to ensure that the magnitude does not affect the comparison. 
 
4. Metric Learning and Optimization:  Instead of training from scratch, the system employs robust pre-
trained models optimized via ONNX Runtime, allowing for hardware-accelerated inference on the edge 
device. Identity verification is performed using Cosine Similarity logic [17] (np.dot), where normalized 
feature vectors are compared against the enrolled database. A similarity threshold is enforced to rigorously 
distinguish between "Authorized" personnel and "Unknown" individuals.  
 
2.5 Embedded implementation and integration (WP#5-6-7) 
 
The physical deployment of the system relies on the Raspberry Pi 5 as the central edge processing unit. Selected 
for its superior performance in AI workloads, the device features a Broadcom BCM2712 Quad-core Arm Cortex-
A76 processor clocked at 2.4GHz and VideoCore VII GPU. This architecture provides the necessary computational 
throughput to run quantized deep learning models (YOLO11n and ArcFace) with minimal latency, significantly 
outperforming previous generations in vector processing tasks. 
 
Video acquisition is handled via the Camera Serial Interface (CSI) rather than standard USB connections to ensure 
high-bandwidth, low-latency streaming directly to the GPU memory. 
To optimize power consumption and prevent thermal throttling during idle periods, the system integrates an HC-
SR04 Ultrasonic Distance Sensor as a hardware trigger. The sensor operates on the sonar principle, emitting a 
40kHz ultrasonic pulse and measuring the time-of-flight for the echo to return. The distance to the subject is 
calculated using the formula:

The high-performance AI pipeline is activated only when the sensor detects a subject within a predefined proximity

2.6 Performance Evaluation (WP#8-9) 
The performance evaluation of the system was conducted in two distinct phases: determining the optimal object 
detection model for edge deployment and validating the accuracy of the facial recognition pipeline using the 
VGGFace2 dataset. 
 
2.6.1 Person Detection Model Selection 
To identify the most suitable model for the embedded system, a comparative analysis was conducted among the 
nano versions of YOLO architectures (v8, v10, 11, and 12). While YOLOv8n achieved the highest average speed 
(21.39 FPS), it exhibited stability issues with significant frame rate drops. Consequently, YOLO11n  was selected 
as the core algorithm; it offered the optimal balance between stability and accuracy, demonstrating the highest 
minimum frame rate (12.20 FPS) and superior detection sensitivity with an average of 14.05 objects per frame.

2.6.2 Face Recognition Accuracy Analysis 
Following the detection stage, the verification performance of the InsightFace (RetinaFace + ArcFace) pipeline was 
evaluated. The test was conducted on a CPU environment using a randomly sampled subset of the VGGFace2 
dataset, consisting of 400 test cases  (100 positive/authorized samples and 300 negative/impostor samples). 
The confusion matrix derived from the test results ( TP: 92, TN: 291, FP: 9, FN: 8 ) indicates a successful 
implementation with expected variance due to environmental factors. The slight drop in precision compared to 
theoretical benchmarks is attributed to the unconstrained nature of the test dataset (e.g., varying poses and lighting).

2.6.3 Deployment Considerations and Expected Trade-offs 
The performance metrics presented above establish the baseline accuracy of the system in an unconstrained 
computational environment. However, the final deployment on the Raspberry Pi edge node requires specific 
optimizations to ensure real-time responsiveness. 
 
 Quantization Impact:  The transition from 32-bit floating-point (FP32) arithmetic to half-precision (FP16) 
or integer (INT8) quantization via ONNX Runtime is expected to significantly reduce latency. While this 
compression is essential for the embedded CPU, it may result in a marginal reduction in recognition 
accuracy (estimated < 1-2%). 
 Optimization Strategy:  The project prioritizes high Precision (avoiding false positives) over Recall. 
Therefore, the optimization thresholds will be tuned to maintain security integrity, even if it requires the 
user to face the camera more directly during the edge operation.
